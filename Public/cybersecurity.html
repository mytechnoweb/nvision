Cybersecurity Fundamentals

Protecting digital assets through awareness, technology, and best practices

Core Concepts

What is Cybersecurity?

Cybersecurity refers to the practice of protecting systems, networks, and data from digital attacks, unauthorized access, and damage. It encompasses a wide range of technologies, processes, and policies designed to safeguard information confidentiality, integrity, and availability. As cyber threats evolve, cybersecurity becomes increasingly critical for individuals, businesses, and governments.

Confidentiality, Integrity, Availability (CIA Triad)

The CIA Triad is a foundational model in cybersecurity: Confidentiality ensures that sensitive information is accessed only by authorized individuals. Integrity guarantees that data remains accurate and unaltered. Availability ensures that systems and data are accessible when needed. Together, these principles guide the design of secure systems and policies.

Authentication and Authorization

Authentication verifies the identity of a user or system, typically through passwords, biometrics, or multi-factor methods. Authorization determines what resources a user can access after authentication. These mechanisms are essential for enforcing access control and preventing unauthorized activity.

Encryption

Encryption is the process of converting data into a coded format to prevent unauthorized access. It uses algorithms and keys to secure information during transmission and storage. Common methods include symmetric encryption (same key for encryption/decryption) and asymmetric encryption (public/private key pairs).

Firewalls

A firewall is a security device or software that monitors and controls incoming and outgoing network traffic based on predefined rules. It acts as a barrier between trusted and untrusted networks, helping to prevent unauthorized access and cyberattacks.

Intrusion Detection and Prevention Systems (IDPS)

IDPS tools monitor network or system activities for malicious actions or policy violations. Intrusion Detection Systems (IDS) alert administrators to suspicious behavior, while Intrusion Prevention Systems (IPS) actively block or mitigate threats in real time.

Threats & Attacks

Malware

Malware is malicious software designed to harm, exploit, or disable computers and networks. Types include viruses, worms, trojans, ransomware, and spyware. Malware can steal data, disrupt operations, or demand ransom payments, making it a major cybersecurity concern.

Phishing

Phishing is a social engineering attack where attackers impersonate trusted entities to trick users into revealing sensitive information, such as passwords or credit card numbers. It often occurs via deceptive emails, websites, or messages.

Denial of Service (DoS) and Distributed DoS (DDoS)

DoS attacks overwhelm a system or network with traffic, rendering it unavailable. DDoS attacks use multiple compromised devices to launch a coordinated flood of traffic. These attacks can disrupt services, damage reputations, and incur financial losses.

Zero-Day Exploits

A zero-day exploit targets a software vulnerability that is unknown to the vendor and has no available patch. These attacks are highly dangerous because they occur before defenses can be implemented. Timely updates and threat intelligence are key to mitigating zero-day risks.

Best Practices

Use Strong Passwords

Create complex passwords using a mix of letters, numbers, and symbols. Avoid using personal information or common words. Use password managers to generate and store secure credentials, and enable multi-factor authentication wherever possible.

Keep Software Updated

Regularly update operating systems, applications, and firmware to patch known vulnerabilities. Enable automatic updates when available, and monitor vendor advisories for critical security releases.

Educate Users

Human error is a leading cause of security breaches. Provide training on recognizing phishing attempts, handling sensitive data, and following security protocols. Encourage a culture of cybersecurity awareness across the organization.

Implement Access Controls

Use role-based access control (RBAC) to limit user permissions based on job responsibilities. Enforce the principle of least privilege, ensuring users have only the access they need to perform their tasks.

Essential Security Commands

Windows

netstat -an – Displays active network connections
tasklist – Lists running processes
systeminfo – Shows system configuration
gpresult /R – Displays group policy results
setx – Sets environment variables
cipher /w – Securely wipes deleted files

Linux

iptables -L – Lists firewall rules
fail2ban-client status – Checks Fail2Ban status
chkrootkit – Scans for rootkits
auditctl -l – Lists audit rules
sudo ufw status – Shows firewall status
who – Displays logged-in users
