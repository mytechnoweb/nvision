System Administration Essentials

Master the backbone of IT infrastructure with key concepts, troubleshooting tips, and command-line tools

Core Concepts

What is System Administration?

System Administration involves managing and maintaining computer systems, servers, and networks to ensure they operate efficiently and securely. It includes tasks such as installing software, configuring hardware, monitoring performance, applying updates, and troubleshooting issues. System administrators play a critical role in ensuring business continuity and IT infrastructure stability.

User and Permission Management

Managing users and permissions is essential for security and access control. This includes creating user accounts, assigning roles, setting file and folder permissions, and enforcing password policies. Tools like Active Directory (Windows) and `passwd`, `chmod`, `chown` (Linux) are commonly used to manage these settings.

Backup and Recovery

Backup and recovery strategies ensure data integrity and availability in case of system failure, data corruption, or cyberattacks. Administrators use automated backup tools, cloud storage, and disaster recovery plans to protect critical data. Regular testing of backups is vital to ensure they can be restored when needed.

Monitoring and Performance

Monitoring tools track system health, resource usage, and performance metrics. Administrators use software like Nagios, Zabbix, or Windows Performance Monitor to detect bottlenecks, memory leaks, or CPU spikes. Proactive monitoring helps prevent downtime and optimize system efficiency.

Patch Management

Patch management involves updating software and operating systems to fix vulnerabilities, improve performance, and add new features. System administrators schedule and test patches to avoid compatibility issues. Tools like WSUS (Windows Server Update Services) and `yum` or `apt` (Linux) are commonly used.

Security and Hardening

Security hardening includes disabling unnecessary services, enforcing strong authentication, applying firewalls, and configuring antivirus software. Administrators also implement logging, auditing, and intrusion detection systems to monitor and protect against threats.

Troubleshooting Techniques

System Boot Issues

If a system fails to boot, check BIOS/UEFI settings, boot order, and hardware connections. Use recovery tools like Windows Recovery Environment or Linux Live CD to access logs and repair boot loaders. Common commands include bootrec /fixmbr and fsck.

Network Connectivity Problems

Start by checking physical connections, IP configuration, and DNS settings. Use commands like ping, ipconfig (Windows), or ifconfig/ip (Linux) to diagnose issues. Firewalls and antivirus software may also block connections.

Slow System Performance

Investigate CPU, memory, and disk usage using tools like Task Manager (Windows) or top/htop (Linux). Check for background processes, malware, or insufficient resources. Disk cleanup and defragmentation may help improve performance.

Application Crashes

Review event logs or system logs for error messages. Ensure the application is compatible with the OS and has necessary permissions. Reinstalling or updating the software often resolves issues. Use Event Viewer (Windows) or journalctl (Linux) for diagnostics.

Important Run Commands

Windows Commands

ipconfig – Displays IP configuration
netstat – Shows network connections
tasklist – Lists running processes
chkdsk – Checks disk for errors
systeminfo – Displays system details
shutdown /r – Restarts the computer
gpupdate /force – Updates group policies

Linux Commands

top – Real-time system monitoring
df -h – Disk space usage
ps aux – Lists all running processes
sudo apt update – Updates package lists
chmod – Changes file permissions
journalctl – Views system logs
systemctl – Manages services
